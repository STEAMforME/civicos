# Contributing to CivicOS

Thank you for your interest in contributing to CivicOS! We're building civic infrastructure for communities everywhere, and we welcome your help.

## Ways to Contribute

### 1. Report Bugs
Found a bug? Please open an issue with:
- Clear description of the problem
- Steps to reproduce
- Expected vs. actual behavior
- Screenshots if applicable

### 2. Suggest Features
Have an idea? Open an issue with:
- Feature description
- Use case (why is this needed?)
- Proposed implementation (if you have ideas)

### 3. Improve Documentation
- Fix typos or unclear instructions
- Add examples
- Translate to other languages

### 4. Write Code
- Fix bugs
- Implement new features
- Improve performance
- Add tests

## Getting Started

### Setup
```bash
# Fork and clone the repo
git clone https://github.com/YOUR_USERNAME/civicos.git
cd civicos

# Create a branch
git checkout -b feature/your-feature-name

# Make your changes
# Test locally by opening index.html

# Commit and push
git add .
git commit -m "Add your feature"
git push origin feature/your-feature-name
```

### Pull Request Process
1. Ensure your code works locally
2. Update README.md if you're adding features
3. Follow the existing code style
4. Write clear commit messages
5. Open a PR with a clear description

## Code Style

### HTML
- Use semantic HTML5 elements
- Indent with 4 spaces
- Include accessibility attributes (alt, aria-label, etc.)

### CSS
- Use CSS variables for colors and spacing
- Mobile-first responsive design
- Comment complex sections

### JavaScript
- Use modern ES6+ syntax
- Add comments for complex logic
- Handle errors gracefully
- Validate user input

## Community Guidelines

- Be respectful and inclusive
- Assume good intentions
- Focus on what's best for the community
- Help others learn and grow

## Questions?

Feel free to:
